LLM 멀티 프로바이더 전환 및 SPARQL 품질 개선

- llm_client.py: LLM_PROVIDER 환경변수로 Gemini/Ollama 전환 지원
- Ollama /api/chat native API 사용 (Qwen thinking 모델 대응)
- 스트리밍 JSONL 파서 구현, Gemini 키 로테이션 로직 보존
- docker-compose.yml: OLLAMA_BASE_URL(host.docker.internal) 등 추가
- .env: LLM_PROVIDER, OLLAMA 관련 설정 추가
- backend/routes.py: 시작 로그에 현재 provider/model 출력
- sparql_tools.py: OPTIONAL 내 FILTER 자동 필수 패턴 변환 후처리
- text2sparql.py: FILTER-OPTIONAL 규칙 및 시간 근접성 예시 추가
- stages.py: target_relation 기반 use_link_prediction 플래그 설정
- supervisor.py: LP는 SPARQL 결과 0건일 때만 트리거
